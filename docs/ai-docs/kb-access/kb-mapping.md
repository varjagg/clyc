# KB mapping

Where [kb-indexing.md](kb-indexing.md) describes the storage shape, **kb-mapping** is the iteration layer on top of it. Its job is to walk one role's bucket of a term's index, applying relevance filters (current-MT, predicate, sense, truth-value, direction, type), and surface assertions to a body of code that knows nothing about index structure. Every inference module, every accessor, every walker that consumes "the assertions about this term in this role" goes through kb-mapping macros.

## When does kb-mapping run?

Always read-only, always per-query. There is no creation event — the mapping layer doesn't own state of its own; it composes existing index data and existing relevance dynamics. Three triggering situations:

1. **An accessor function is called.** `pred-values term pred 1 2` ([`kb-mapping-utilities.lisp:170`](../../../larkc-cycl/kb-mapping-utilities.lisp#L170)) — "give me the arg2 values for every gaf where `term` is at arg1 and `pred` is the predicate" — opens an iteration over the `:gaf-arg/1/pred` role of `term`'s index.

2. **An inference module asks for candidate assertions.** Removal modules call `do-gaf-arg-index`, `do-predicate-rule-index`, `do-isa-rule-index`, etc. with the bound query terms. Each candidate then goes through the module's per-assertion check.

3. **A bulk-removal cascade gathers assertions.** `gather-<role>-index` ([`kb-mapping.lisp:303-919`](../../../larkc-cycl/kb-mapping.lisp#L303)) consumes an entire role into a list. `remove-term-indices` ([`kb-indexing.lisp:627`](../../../larkc-cycl/kb-indexing.lisp#L627)) collects `gather-other-index`, `gather-mt-index`, every `gather-<rule-role>-index` for both senses, and `gather-gaf-arg-index` per argnum, then TMS-removes each batch.

The iteration is **bounded in time** — the moment the body returns or `*mapping-done*` is thrown, the iterator is destroyed and resources reclaimed. There is no observable footprint after the call.

## The two layers

Each role has a pair of operators:

- A **`do-<role>-index`** iteration macro for streaming use ("apply this body to every assertion"). Keeps no list, supports early termination via `:done`.
- A **`gather-<role>-index`** function for collection use ("give me the list"). Always eagerly consumes the iterator and `fast-delete-duplicates`.

Both are built on the same underlying machinery — a **final-index-spec iterator** generated by `define-kb-final-index-spec-iterator` ([`kb-mapping-macros.lisp:148`](../../../larkc-cycl/kb-mapping-macros.lisp#L148)).

## Final-index-spec iterators

A *final-index-spec* is a path through the indexing tree down to a single leaf-set: e.g. `(term :gaf-arg argnum predicate mt)`. Walking a role with sub-keys means generating every relevant final-index-spec, then for each spec walking the leaf-set under it.

`define-kb-final-index-spec-iterator name lambda-list :slots ... :singleton-form ... :done-form ... :next-form ... :relevant-keylist ... :quiesce-step ...` declares one such iterator. The macro generates:

- A state struct with the named slots.
- `new-<name>-final-index-spec-iterator` — constructor. If the term's index is currently simple (small-N flat list), returns a `new-singleton-iterator` over the role's `:singleton-form` (which is a `(term :simple ...)` final-index-spec for the simple-index assertion-list with the role's `matches-<role>-index` predicate as filter). Otherwise initializes the per-key state and returns a generic `new-iterator` whose advance function is the generated `<name>-final-index-spec-iterator-next`.
- `<name>-final-index-spec-iterator-quiesce` — the state machine that walks the keys until the current keylist is both **valid** (no nil keys) and **relevant** (`relevant-pred?`, `relevant-mt?`, etc.). Quiescing pops irrelevant keys and refills empty key slots from `key-<role>-index` calls.
- `<name>-final-index-spec-iterator-next` — calls quiesce, then `next-form` (which advances the state to expose the next final-index-spec).

The generated state is per-key-level: e.g. for `:gaf-arg` the slots are `(term predicate note argnum-keys predicate-keys mt-keys)`. Each level holds a list of the remaining keys at that level — the outer iteration pops from `argnum-keys`, refills `predicate-keys` from the next argnum's predicate set, refills `mt-keys` from that argnum/predicate's MT set, and so on. The state lives in a struct (vector storage) for fast access.

The `:relevant-keylist` form is the key combiner — it returns a complete keylist `(argnum pred mt)` if the current state is valid+relevant, else `nil` to trigger another quiesce step.

The `:quiesce-step` form is one "pop one bad key and try again" step. It returns nil to keep going, or t when there are no more keys at any level (the iterator is exhausted).

The instances:

| Role | `define-kb-final-index-spec-iterator` parameters | Refilling helpers |
|---|---|---|
| `gaf-arg` | `(term &optional argnum predicate)` | `gaf-arg-…-refill-predicate-keys` / `…-refill-mt-keys` |
| `predicate-extent` | `(predicate)` | (none — single mt-keys list) |
| `nart-arg` | `(term &optional argnum function)` | partial — `next-form` is `missing-larkc 30408` |
| `function-extent` | `(function)` | singleton-only |
| `predicate-rule` | `(pred sense &optional direction)` | `…-refill-mt-keys`, `…-refill-direction-keys` |
| `decontextualized-ist-predicate-rule` | `(pred sense &optional direction)` | `…-refill-direction-keys` |
| `isa-rule` | `(col sense &optional direction)` | (mt and direction keys, refills via shared helpers) |
| `quoted-isa-rule` | `(col sense &optional direction)` | same |
| `genls-rule` | `(col sense &optional direction)` | same |
| `genl-mt-rule` | `(genl-mt sense &optional direction)` | same |
| `pred-arg2-rule` | `(pred arg2 &optional sense direction)` | `…-refill-mt-keys`, `…-refill-direction-keys` |
| `function-rule` | `(function &optional direction)` | mt + direction |
| `exception-rule` | `(rule &optional direction)` | mt + direction |
| `pragma-rule` | `(rule &optional direction)` | mt + direction |
| `unbound-predicate-rule` | `(&optional sense direction)` | (no term — auxiliary index) |

The `mt`, `other` roles are simpler — they always emit one final-index-spec (the term-level `(term :ist)` or `(term :other)`), so they don't go through `define-kb-final-index-spec-iterator`. They use `mt-final-index-spec` / `other-final-index-spec` directly.

`new-final-index-iterator final-index-spec type truth direction` is the second-level iterator. It walks the leaf-set under one final-index-spec, filtered by:

- `type` — `:gaf` keeps only GAFs, `:rule` keeps only rules, `nil` keeps both.
- `truth` — keeps only assertions with this truth value.
- `direction` — keeps only assertions with this direction.

For simple-index final-index-specs `(term :simple match-fn)`, the iterator walks the term's flat assertion list and applies `match-fn` to filter to the role-relevant ones (since the simple index doesn't partition by role).

`destroy-final-index-iterator` releases iterator state. Always paired with `unwind-protect` in macro expansions — the iterator may hold references that need to be cleared.

## The do-\\<role\\>-index macros

Every role has a `do-<role>-index` macro. They all expand to the same shape: outer `until` loop on the final-index-spec iterator, inner `until` loop on `new-final-index-iterator`, body on each yielded assertion. The inner loop wraps `unwind-protect` around `destroy-final-index-iterator` so an early `:done` exit cleans up.

Sample (gaf-arg):

```lisp
(defmacro do-gaf-arg-index ((var term &key index predicate truth direction done) &body body)
  ;; Walk every gaf assertion where TERM is the indexnum-th argument
  ;; and (if specified) PREDICATE is the predicate.
  ...)
```

Keyword parameters per macro:

| Macro | Term inputs | Filters |
|---|---|---|
| `do-gaf-arg-index (var term ...)` | `:index argnum`, `:predicate pred` | `:truth`, `:direction`, `:done` |
| `do-gaf-arg-index-values (var term gather-index ...)` | same | same; binds `var` to the *gather-index*-th arg of each assertion |
| `do-predicate-extent-index (var pred ...)` | — | `:truth`, `:direction`, `:done` |
| `do-nart-arg-index (var term ...)` | `:index argnum`, `:function func` | `:done` (no truth/direction support) |
| `do-function-extent-index (var func ...)` | — | `:done` |
| `do-predicate-rule-index (var pred ...)` | — | `:sense`, `:direction`, `:done` |
| `do-decontextualized-ist-predicate-rule-index (var pred ...)` | — | `:sense`, `:direction`, `:done` |
| `do-isa-rule-index (var col ...)` | — | `:sense`, `:direction`, `:done` |
| `do-quoted-isa-rule-index (var col ...)` | — | `:sense`, `:direction`, `:done` |
| `do-genls-rule-index (var col ...)` | — | same |
| `do-genl-mt-rule-index (var genl-mt ...)` | — | same |
| `do-function-rule-index (var func ...)` | — | `:direction`, `:done` |
| `do-exception-rule-index (var rule ...)` | — | same |
| `do-pragma-rule-index (var rule ...)` | — | same |
| `do-unbound-predicate-rule-index (var ...)` | — | `:sense`, `:direction`, `:done` |
| `do-mt-index (var mt ...)` | — | `:type`, `:truth`, `:direction`, `:done` |
| `do-broad-mt-index (var mt ...)` | — | `:type`, `:truth`, `:done` (mostly missing-larkc) |
| `do-mt-contents (var mt ...)` | — | dispatches between `:mt-index` and `:broad-mt-index` |
| `do-other-index (var term ...)` | — | `:type`, `:truth`, `:direction`, `:done` (body is missing-larkc) |
| `do-term-index (var term ...)` | — | `:type`, `:truth`, `:direction`, `:done` — walks every role |
| `do-overlap-index (var terms ...)` | — | `:truth`, `:done` (missing-larkc) |
| `do-best-gaf-lookup-index (var asent ...)` | `:methods` | `:truth`, `:done` (missing-larkc) |
| `do-gaf-lookup-index (var lookup-index ...)` | — | `:truth`, `:done` (missing-larkc) |
| `do-best-nat-lookup-index`, `do-nat-lookup-index` | NAT versions of the GAF planners | (missing-larkc) |
| `do-final-index-from-spec (var fis ...)` | — | low-level: walks one specific final-index-spec |

`do-term-index` is the union over all role iterators: it builds an iterator-of-iterators (`new-iterator-iterator`) by pushing every role's `new-<role>-final-index-spec-iterator` for the term, then walks them in sequence. Used by `gather-index` to collect every assertion mentioning a term. The `:other` final-index-spec gets post-hoc semantic filtering via `do-term-index-assertion-match-p` since `:other` is the catch-all and may contain things the role-keyed iterators didn't cover.

## The pwhen filter macros

Inside the body of a `do-<role>-index` form, several PWHEN-prefixed macros guard sub-iterations. They're declarative one-liners that wrap a `when` around a body:

| Macro | Guard |
|---|---|
| `pwhen-predicate-is-relevant pred body` | `(relevant-pred? pred)` |
| `pwhen-mt-is-relevant mt body` | `(relevant-mt? mt)` |
| `pwhen-truth-relevant truth truth-spec body` | `(truth-relevant-p truth truth-spec)` — null spec ≡ any |
| `pwhen-assertion-has-truth assertion truth body` | `(or (null truth) (assertion-has-truth assertion truth))` |
| `pwhen-assertion-has-direction-spec assertion dir-spec body` | `(or (null dir-spec) (eq dir-spec (assertion-direction assertion)))` |
| `pwhen-assertion-has-type assertion type body` | `(or (null type) (case type (:gaf …) (:rule …) (t …)))` |

These are the building blocks for the more compositional iteration patterns: e.g. `do-assertion-list` is `(do-list … (pwhen-assertion-has-truth … (pwhen-assertion-has-direction-spec … body)))`, and `do-simple-index` is `do-assertion-list` over the simple-index assertion list.

## Map-* and gather-* functions

For each role there's a pair:

- `map-<role>-index function term ...` — apply `function` to every assertion matching the role's keys, with current-MT relevance.
- `gather-<role>-index term ...` — collect into a list, then `fast-delete-duplicates` by `eq`.

Both wrap a `do-<role>-index` iteration and use `catch :mapping-done` so a body throwing that tag terminates early. The map-* variants honor `*mapping-done*`-throwing termination via the `mapping-finished` helper. The gather-* variants always run to completion.

`map-mt-contents` ([`kb-mapping.lisp:197`](../../../larkc-cycl/kb-mapping.lisp#L197)) is special: when the MT is broad (`broad-mt?` per [microtheories.md](../core-kb/microtheories.md)), there is no per-MT subindex (broad MTs are excluded — every assertion would pile in there). Instead it walks **every assertion in the KB** via `do-assertions-table`, with a percent-progress note. For non-broad MTs it delegates to `map-mt-index`.

`gather-index term` ([`kb-mapping.lisp:303`](../../../larkc-cycl/kb-mapping.lisp#L303)) is the catch-all: walks every role of `term`'s index via `do-term-index`. `gather-index-in-any-mt` rebinds the relevance dynamics to "everything" before delegating.

## Higher-level accessor patterns

`kb-mapping-utilities.lisp` is a sheaf of common query patterns built on `kmu-do-index-iteration` (a local iteration macro that wraps `do-gaf-arg-index`-style iteration):

| Pattern | Returns | Description |
|---|---|---|
| `some-pred-value term pred [arg] [truth]` | boolean | Does any assertion satisfy `(pred ?... term ...)` at index-arg? |
| `some-pred-value-if term pred test [arg] [truth]` | boolean | Same, with an extra per-assertion `test` filter |
| `fpred-value-gaf term pred [arg] [truth]` | assertion | First matching assertion |
| `fpred-value term pred [iarg] [garg] [truth]` | term | The `garg`-th arg of the first matching assertion |
| `pred-values term pred [iarg] [garg] [truth]` | list | All `garg`-th args from matching assertions |
| `pred-u-v-holds pred u v [u-arg] [v-arg] [truth]` | boolean | Does `(pred u v)` (with arg positions) hold? |
| `pred-arg-values term pred arg [tpsn] [apsn] [gpsn] [truth]` | list | Three-way join: `term` at tpsn, `arg` at apsn, gather gpsn |
| `pred-value-tuples term pred iarg gargs [truth]` | list of tuples | Tuple of multiple gather positions |

Each comes in three variants:

- *Default*: respects ambient `*mt*` / `*relevant-mt-function*`.
- *-in-any-mt*: rebinds to "everything" (walks all MTs).
- *-in-relevant-mts*: takes an MT argument and either walks all MTs (if nil) or all `genlMts` of the given MT.

`gp-map-arg-index` ([`kb-gp-mapping.lisp:45`](../../../larkc-cycl/kb-gp-mapping.lisp#L45)) is a specialized version that walks not just `pred` but every spec-pred of `pred` (binding `*relevant-pred-function*` to `relevant-pred-is-spec-pred`). Plus, if `pred` is binary, it walks the inverse direction (binding to `relevant-pred-is-spec-inverse` and swapping arg positions). This is how queries lift through `genlPreds`/`genlInverse` reasoning. The arg-swap is observable in the body via `*mapping-arg-swap*`.

## Dynamic state

The mapping layer reads ambient dynamic state established by callers:

| Variable | Bound by | Read by |
|---|---|---|
| `*mt*`, `*relevant-mt-function*` | callers (`(let ((*mt* …)) …)` or `possibly-with-just-mt`) | `relevant-mt?` inside iterator quiesce |
| `*pred*`, `*relevant-pred-function*` | callers (gp-map-arg-index, etc.) | `relevant-pred?` |
| `*mapping-done*` | `(catch :mapping-done …)` | `(throw :mapping-done …)` in body, also `mapping-finished` helper |
| `*mapping-answer*` | `gather-mt-index`, `gather-other-index` (callers bind, body pushes) | result accumulator pattern |
| `*mapping-target*` | `find-rule-cnf-via-index-int` (caller binds) | `find-cnf-internal` matches against this |
| `*candidate-assertion*` | iterator body | `*cnf-matching-predicate*` / `*gaf-matching-predicate*` use this for normalization |
| `*mapping-equality-test*` | callers (e.g. when arg values may be HLMTs requiring `equal`) | `pushnew` test in pred-values* |
| `*mapping-assertion-bookkeeping-fn*` | callers (e.g. inference instrumentation) | called per yielded assertion |
| `*mapping-assertion-selection-fn*` | `gather-assertions` callers | filter applied before pushing into `*mapping-answer*` |
| `*mapping-truth*`, `*mapping-direction*` | `map-<role>-index` callers | unused in body but tracked for future |
| `*mapping-arg-swap*` | `gp-map-arg-index` | body may inspect to know whether arg1/arg2 were swapped |
| `*map-term-selective-test*`, `*map-term-selective-action*` | (mostly unused — staging for selective term iteration) | — |

## Notes for a clean rewrite

- **Replace `define-kb-final-index-spec-iterator` with a generator over a declarative role schema.** Once the role taxonomy is unified ([kb-indexing.md](kb-indexing.md) "Notes for a clean rewrite"), the iterator state machine is mechanical: walk each level's keys filtering by relevance. A single generic iterator parameterized by the role schema replaces ~14 nearly-identical iterator definitions and ~14 `do-<role>-index` macros.
- **The `do-<role>-index` macro expansion is enormous and identical-shape across roles.** Each generated macro is ~30 lines of `until`/`unwind-protect`/`new-final-index-iterator` plumbing. A modern design has one `do-final-indices` that takes a `(role term &key sub-keys)` description and runs the same nested loop.
- **`kmu-do-index-iteration` is the local stopgap macro the port uses to collapse the boilerplate** — its existence is evidence the user already wanted the unified iteration form.
- **The simple-vs-complex split bleeds into the iterator.** Every role's iterator constructor checks `simple-indexed-term-p` and dispatches. The clean rewrite removes the simple form (per kb-indexing.md note), eliminating the singleton-form / `matches-<role>-index` post-hoc filter path entirely.
- **Broad-MT iteration walks the entire assertion table.** This is unavoidably O(N) but only because broad MTs are excluded from the per-MT subindex by design. A clean rewrite could either (a) include broad MTs in their own dedicated index, paying the storage cost for fast lookups, or (b) keep them out and accept the linear scan, but provide a structured incremental walk with progress reporting (the port has `noting-percent-progress`, currently combined with `missing-larkc 9466`/`9467` for the actual body work — these are TMS-removal helpers).
- **`*mapping-*` dynamic variables are a poor man's iterator parameter list.** Modern design passes them explicitly. The bookkeeping-fn, equality-test, and selection-fn are all natural keyword arguments.
- **`gp-map-arg-index` lives in its own file (`kb-gp-mapping.lisp`) but is an integral part of the kb-mapping API.** The split is historical — the file was probably introduced after kb-mapping was already locked. A clean rewrite folds it back in.
- **`do-overlap-index`, `do-best-gaf-lookup-index`, `do-gaf-lookup-index`, `do-best-nat-lookup-index`, `do-nat-lookup-index` are all missing-larkc.** The clean rewrite needs these — they are the macros that actually run the planner-produced lookup-index, dispatching by `:index-type` to the right per-role iterator. Without them, `find-gaf-formula` had to inline ~150 lines of plumbing.
- **The post-hoc semantic filter for `:other` (`do-term-index-assertion-match-p`) is the only role with this need.** It's a smell that `:other` is the catch-all bucket that accumulates terms that didn't fit the role-keyed iterators — modeling it as a graph index instead would let every term be indexed under every role it actually plays, making the `:other` filter unnecessary.
- **`pred-arg-values-in-relevant-mts` is `missing-function-implementation`** — an exact copy-of-`pred-arg-values` wrapped in `possibly-in-mt` would be enough; nothing in LarKC supplied it.
